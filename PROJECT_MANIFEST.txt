================================================================================
SIGNALING PROTOCOL (11 EVENT TYPES)
================================================================================

Event Name         | Direction      | Payload                  | Purpose
─────────────────────────────────────────────────────────────────────────
register           | Client→Server  | {peerId}                 | Register peer
call-request       | Client→Server  | {from, to}               | Initiate call
incoming-call      | Server→Client  | {from, to}               | Notify callee
call-accept        | Client→Server  | {from, to}               | Accept call
call-accepted      | Server→Client  | {from, to}               | Confirm accept
call-reject        | Client→Server  | {from, to}               | Reject call
call-rejected      | Server→Client  | {from, to}               | Confirm reject
offer              | Client→Server  | {from, to, sdp}          | Send SDP offer
answer             | Client→Server  | {from, to, sdp}          | Send SDP answer
ice-candidate      | Client→Server  | {from, to, candidate}    | Send ICE candidate
hang-up            | Client→Server  | {from, to}               | End call

================================================================================
CALL STATE MACHINE
================================================================================

Caller Side:
  idle
    → (user types remote ID and clicks Call)
  calling
    → (receive call-accepted from callee)
  connecting
    → (SDP exchange + ICE gathering)
  connected
    → (iceConnectionState = connected)
  disconnected (2s timeout)
    → idle

Callee Side:
  idle
    → (receive incoming-call)
  ringing
    → (user clicks Accept or Reject)
  connecting (if Accept clicked)
    → (receive offer from caller, send answer)
  connected
    → (iceConnectionState = connected)
  disconnected (2s timeout)
    → idle

HOW TO RUN
================================================================================

DEVELOPMENT:
  1. cd jwan-call
  2. npm install
  3. npm run dev
  4. Open http://localhost:5173 in two browser tabs
  5. Grant microphone permission
  6. Copy peer ID from Tab 1, paste into Tab 2
  7. Click Call and Accept

PRODUCTION:
  1. npm run build --workspace=client
  2. Copy client/dist/* to web server
  3. Run server: NODE_ENV=production node server/src/index.js
  4. Proxy /socket.io to server via nginx/Cloudflare

================================================================================
LIMITATIONS & DISCLAIMERS
================================================================================

1. NO TURN SERVER
   • Uses only Google STUN servers
   • May fail on strict NAT/corporate firewalls
   • Same LAN and simple NAT work fine
   • Production deployment needs TURN server

2. NO PERSISTENCE
   • Peer IDs are ephemeral (10 random chars)
   • No call history logged
   • No user accounts or authentication
   • Session data lost on page reload

3. NO MULTI-PARTY
   • One-on-one calls only
   • No conference/group calling
   • Multiple sequential calls OK

4. AUDIO ONLY
   • WebRTC configured for audio only
   • No video support (easy to add)
   • No screen sharing

5. SINGLE SERVER
   • Not horizontally scalable without state sharing
   • Would need Redis adapter for multi-server setup
   • Suitable for ~1000 concurrent users per server

================================================================================
SUPPORT & TROUBLESHOOTING
================================================================================

"Microphone permission denied"
  → Grant permission in browser settings
  → Reload page

"Can't connect to server"
  → Verify npm run dev started both servers
  → Check ports 3001 and 5173 are available
  → Check firewall settings

"No audio heard after connecting"
  → Verify microphone is working (test in another app)
  → Confirm "Connected" status in UI
  → Check browser console for WebRTC errors
  → Use different devices for caller/callee (same device hearing self)

"Call fails when on different networks"
  → TURN server needed (deploy coturn or use service)
  → Edit server/src/index.js RTC_CONFIG to add TURN
